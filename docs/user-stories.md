# User Stories — team-tasks

## 1. 핵심 시나리오

**페르소나**: 팀원 김지수 (일반 직원, 마케팅팀 3년차)  
**목표**: 출근부터 퇴근까지 오늘 할 일을 놓치지 않고 처리한다

---

### 09:00 출근 — 오늘 할 일 파악

> **As a** 팀원  
> **I want to** 로그인 후 '내 업무' 목록을 우선순위 순으로 바로 볼 수 있어서  
> **So that** 어떤 일부터 시작할지 5초 안에 판단할 수 있다

- 앱을 열면 기본 화면이 **내 업무 목록** (할 일·진행 중만 필터된 상태)
- 마감일이 오늘인 업무에 강조 표시가 붙어 있어 한눈에 식별 가능
- 어젯밤 팀 리더가 새로 배정한 업무 1건에 **신규** 배지가 표시됨

---

### 09:10 업무 시작 — 상태 변경

> **As a** 팀원  
> **I want to** 업무를 탭 한 번으로 "할 일 → 진행 중"으로 바꿀 수 있어서  
> **So that** 팀 리더가 실시간으로 진행 여부를 알 수 있다

- 오늘 첫 번째 업무 카드를 클릭 → 상태 셀렉터에서 **진행 중** 선택 → 저장
- 칸반 보드에서는 카드가 "할 일" 열 → "진행 중" 열로 이동

---

### 11:30 중간 점검 — 메모 추가

> **As a** 팀원  
> **I want to** 업무 카드 안에 짧은 진행 메모를 남길 수 있어서  
> **So that** 팀 리더가 별도 메신저 없이 현황을 파악할 수 있다

- 업무 상세를 열어 메모 필드에 "초안 작성 완료, 검토 대기 중" 입력 후 저장
- 저장 즉시 목록 화면으로 돌아옴 (페이지 이동 없이)

---

### 13:00 점심 후 — 새 배정 업무 확인

> **As a** 팀원  
> **I want to** 점심 시간 중 새로 들어온 업무를 알림 없이도 쉽게 확인할 수 있어서  
> **So that** 긴급 업무를 놓치지 않는다

- 목록 상단에 **오늘 배정됨** 섹션이 분리되어 표시
- 우선순위 "높음"인 업무가 새로 1건 배정된 것을 확인
- 기존 진행 중 업무와 비교해 순서를 머릿속으로 재조정

---

### 15:00 업무 완료 — 상태 닫기

> **As a** 팀원  
> **I want to** 완료한 업무를 "완료"로 표시하는 데 클릭 2회를 넘기지 않아서  
> **So that** 귀찮아서 미루다 상태가 틀어지는 일이 없다

- 업무 카드 우측 체크 버튼 클릭 → 상태 자동으로 **완료** 전환
- 완료된 카드는 목록 하단으로 내려가거나 흐리게 표시되어 시각적 노이즈 감소

---

### 17:30 퇴근 전 — 미완료 업무 정리

> **As a** 팀원  
> **I want to** 오늘 미완료로 남은 업무를 한 번에 확인하고 메모를 남길 수 있어서  
> **So that** 내일 아침 인수인계 없이 바로 이어서 작업할 수 있다

- 필터: **오늘 마감 + 미완료** → 업무 1건 노출
- 해당 카드에 "내일 오전 중 마무리 예정" 메모 추가 후 저장
- 앱 종료

---

## 2. 결정 사항

| # | 질문 | 결정이 필요한 이유 |
|---|------|------------------|
| 1 | 담당자(팀원)도 업무 상태를 직접 변경할 수 있는가, 아니면 팀 리더만 변경할 수 있는가? | 시나리오에서 팀원이 "할 일 → 진행 중 → 완료"를 스스로 바꾸지만, 권한을 팀 리더로 제한할 경우 UI 흐름 전체가 달라진다 |
| 2 | 메모는 1건(최신 1개 덮어쓰기)인가, 아니면 타임스탬프가 붙은 댓글 형태로 누적되는가? | 단일 메모 필드면 DB 컬럼 하나로 충분하지만, 댓글 누적이면 별도 테이블이 필요해 MVP 범위를 벗어날 수 있다 |
| 3 | '신규 배정' 배지는 담당자가 카드를 열면 자동으로 사라지는가, 아니면 별도 확인 액션이 필요한가? | 읽음 처리 여부에 따라 `is_read` 컬럼 추가와 업데이트 로직이 생기며 복잡도가 올라간다 |
| 4 | 업무의 마감일을 담당자가 직접 수정할 수 있는가, 아니면 팀 리더만 수정 가능한가? | 퇴근 전 미완료 처리 시나리오에서 팀원이 마감일을 자의로 밀어버리면 배정 의도가 희석된다 |
| 5 | 완료된 업무는 목록에서 즉시 숨김 처리되는가, 아니면 흐리게 남겨두고 명시적으로 '완료 보기' 토글로 전환하는가? | 즉시 숨김이면 UX가 단순하지만, 오늘 완료한 일을 퇴근 전에 다시 확인하려는 시나리오(17:30)와 충돌한다 |

---

### 시나리오 요약 — MVP에서 필요한 기능

| # | 기능 | 트리거 시점 |
|---|------|------------|
| 1 | 내 업무 목록 조회 (우선순위·마감일 정렬) | 출근 |
| 2 | 업무 상태 변경 (할 일 / 진행 중 / 완료) | 업무 시작·완료 시 |
| 3 | 업무 메모 입력·수정 | 중간 점검·퇴근 전 |
| 4 | 신규 배정 업무 식별 | 점심 후 |
| 5 | 미완료 필터 | 퇴근 전 |
